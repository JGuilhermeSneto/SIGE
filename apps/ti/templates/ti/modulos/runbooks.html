Incidente: CPU em 100%

Procedimento para identificar e mitigar picos de processamento anormais.

CRÍTICO

1. Diagnóstico Inicial

Acesse o terminal do servidor e verifique quais processos estão consumindo mais recursos utilizando o comando htop ou top.

# Verificando processos Python/Gunicorn
ps aux | grep python | sort -rk 3,3 | head -n 5

2. Mitigação

Se o problema for causado por uma tarefa longa no Celery, você pode tentar purgar a fila se não houver dados críticos pendentes.

# PERIGO: Isso limpará TODAS as tarefas pendentes
celery -A config purge

3. Prevenção

  • Verifique se há loops infinitos em middlewares recentemente alterados.
  • Confirme se o cache do Redis está operacional.
  • Considere o escalonamento horizontal (adicionar mais workers).
