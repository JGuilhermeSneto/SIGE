## 🚀 Como Usar (Para Dev)

### Em JavaScript
```javascript
// Trocar tema programaticamente
document.documentElement.setAttribute('data-theme', 'terra-musgo');

// Ler tema atual
const current = document.documentElement.getAttribute('data-theme');
console.log(current); // "terra-musgo"

// Salvar preferência (automático via theme_manager.js)
localStorage.setItem('sige_theme', 'terra-musgo');
```

### Em CSS
```css
/* Regras que mudam por tema */
html[data-theme="brisa-aqua"] .sige-card {
  border-radius: 28px;
  background: linear-gradient(180deg, rgba(224,255,255,0.78), rgba(175,238,238,0.55));
  backdrop-filter: blur(14px);
}

html[data-theme="cinza-industrial"] .sige-card {
  border-radius: 6px;
  background: transparent;
  box-shadow: var(--shadow-sm);
}
```

### Em HTML
```html
<!-- Automático, você não precisa fazer nada! -->
<html data-theme="indigo-profundo">
  <!-- CSS vars já estão aplicadas -->
</html>
```

## 🧪 Como Testar Cada Tema

1. Abra o site em um navegador
2. Clique no ícone paleta (🎨) no topbar
3. Selecione um tema da lista
4. Observe:
   - ✅ Cores mudam suavemente
   - ✅ Cards têm nova aparência
   - ✅ Calendários mais legíveis
   - ✅ Animações diferentes
   - ✅ Bordas diferente radius
   - ✅ Fontes (se aplicável)
5. Recarregue a página — tema persiste!

## 🎓 Design Thinking

Cada tema foi criado com:

1. **Propósito Claro** — O que transmite? (conforto, eficiência, premium, etc)
2. **Paleta Coerente** — Mínimo 6 tons conectados
3. **Hierarquia Visual** — Contraste suficiente (WCAG AA)
4. **Motion Signature** — Animação que reflete o tema
5. **Usabilidade** — Cards/calendários sempre legíveis
6. **Consistência** — Mesmas variáveis CSS em todos
